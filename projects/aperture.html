aperture

A visual debugger for AI agent traces. Drop in a run, see its shape as a tree and a timeline, and replay it step by step — no backend.

When an agent does something weird — wrong tool, runaway loop, hallucinated argument — all you usually have is a JSON dump of the run: thousands of lines, deeply nested, hard to scan. Aperture renders that JSON as a tree and a timeline. You see the shape of the run at a glance, then click into any span for its prompt, response, and tool I/O. An agent tells a customer their order is canceled when it isn't? The tree shows two lookup_order calls; the assistant turn between them reveals the model misread the order ID the second time. About ninety seconds, instead of scrolling for fifteen minutes.

See it work

A ~90-second walkthrough: dropping a trace, navigating the tree and timeline, searching and filtering, and replaying the run span by span.

How it works

Aperture is local-first: drop a trace JSON file on the page and three views light up from the same selection.

  • Tree, timeline, and detail — one selection. The tree shows the call hierarchy, the timeline shows when each span ran and how long it took, and the detail panel shows the selected span's content blocks — thinking, tool calls, prompts, and results. Click anywhere and all three stay in sync.
  • Search and filter, like a real debugger. Type a query and non-matching spans dim while matches highlight and their ancestor path auto-expands; toggle "errors only" to collapse the run down to just what failed.
  • Subagent boundaries. Delegated work renders as its own span on the timeline, so a subagent's input and result read as a unit instead of disappearing into the parent's stream.
  • Step-through replay. Reset to the start and play the run forward span by span — the tree expands and scrolls, the detail panel narrates each turn. Pause on any step, jump elsewhere, and resume from there. Agent execution becomes a navigable sequence.
  • Concurrency without overlap. Parallel spans pack into sub-lanes on the timeline, so four simultaneous tool calls render side by side instead of stacking on top of each other.

Bring your own trace format

Aperture is built for traces from the Claude Agent SDK, but the internal schema is small and documented. Any other harness can be ingested with a short adapter — a ~30-line function that maps your events to the Span/Trace shape — with no changes to the app itself, since it validates any object matching the schema on drop.

Stack

Vite + React 19 + TypeScript, with Zustand for app state and Zod for schema validation. The timeline canvas uses Three.js and GSAP. There's no backend — everything runs in the browser, and a trace never leaves your machine.
